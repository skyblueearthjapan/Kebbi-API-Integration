**作成日**: 2026年3月18日
**更新日**: 2026年3月18日（v2 — 設計判断確定）
**対象システム**: NUWA RoFlow 受付プロジェクト（株式会社ラインワークス 新工場事務所）
**目的**: GAS開発者への実装指示書

## 1. システム概要

NUWA受付ロボットが来客対応を行う。来客が「打合せによる訪問」「社内スタッフに用事」「その他のご用件」を選択した後、GAS API経由で会議室予約情報やスタッフ情報を取得し、ロボットが音声で案内する。

### 1.1 全体アーキテクチャ（確定）

```
                          ┌─────────────────────────────────────┐
                          │        スプレッドシート群             │
                          │                                     │
                          │  [会議室予約シート]  [スタッフ管理シート] │
                          │        ↑ 読み書き      ↑ 読み書き     │
                          └───┬──────────────┬──────────────────┘
                              │              │
                   ┌──────────┘              └──────────┐
                   │                                     │
            [GAS-A: 会議室予約アプリ]            [GAS-C: スタッフ管理アプリ]
            （既存・Web UI）                     （既存・Web UI）
            ID: 11cxEtE_...                     ID: 12VI1i4TK3D...

                          ┌─────────────────────┐
                          │  GAS-B（新規作成）    │
                          │  ロボット専用API      │
                          │  読み取り専用で参照    │
                          └──────────┬──────────┘
                                     │
                              HTTP GET / テキスト応答
                                     │
                          ┌──────────┴──────────┐
                          │  NUWAロボット (RoFlow) │
                          └─────────────────────┘
```

**設計判断（B案採用）**:
- GAS-A（会議室予約）、GAS-C（スタッフ管理）は既存のまま変更なし
- GAS-B（新規）がロボット専用APIとして、両方のスプレッドシートを読み取り専用で参照
- 既存アプリに影響を与えない安全な構成

## 2. GAS-B 実装仕様（新規作成）

### 2.1 概要

| 項目 | 値 |
|---|---|
| プロジェクト種別 | スタンドアロン GAS プロジェクト（新規作成） |
| エントリポイント | `doGet(e)` |
| レスポンス形式 | プレーンテキスト（`ContentService.createTextOutput()`） |
| 実行者 | 自分（スプレッドシートへのアクセス権限を持つアカウント） |
| アクセス権限 | 全員（Anyone）— NUWAロボットが認証なしでアクセス |

### 2.2 スクリプトプロパティ（GAS-Bに設定）

| プロパティ名 | 説明 | 例 |
|---|---|---|
| `RESERVATION_SPREADSHEET_ID` | 会議室予約アプリのスプレッドシートID | `1ABC...xyz` |
| `STAFF_SPREADSHEET_ID` | スタッフ管理アプリのスプレッドシートID | `1DEF...uvw` |
| `SOUMU_EXTENSION` | 総務の代表内線番号 | `1000` |

### 2.3 doGet ルーティング

```javascript
function doGet(e) {
  var action = e.parameter.action;

  if (action === "getTodayReservations") {
    return getTodayReservations();
  }
  if (action === "getStaffInfo") {
    return getStaffInfo(e.parameter.staffName);
  }
  if (action === "getStaffList") {
    return getStaffList();
  }

  return ContentService.createTextOutput("不明なアクションです");
}
```

## 3. API #1: 会議室予約取得 — `getTodayReservations`

### 3.1 リクエスト

```
GET /exec?action=getTodayReservations
```

### 3.2 参照先スプレッドシート（会議室予約アプリのデータ）

GAS-Bから `SpreadsheetApp.openById(RESERVATION_SPREADSHEET_ID)` で読み取り。

**シート名**: `予約受付リスト`
**ヘッダー行**: 2行目
**データ行**: 3行目以降

| ヘッダー名 | 内部キー | 用途 |
|---|---|---|
| 予約ID | `reservationId` | 識別用 |
| 日付 | `date` | 本日フィルタ用（yyyy-MM-dd） |
| 開始時間 | `startTime` | 時間案内（HH:mm） |
| 終了時間 | `endTime` | 時間案内（HH:mm） |
| 会議室名 | `room` | 会議室案内 |
| 名前 | `name` | 予約者名 |
| お客様名 | `customerName` | 来訪者名 |
| 打合せ内容 | `meetingDetail` | 補足情報 |
| ステータス | `status` | `active` のみ対象 |

### 3.3 処理ロジック

1. 本日の日付を取得（`yyyy-MM-dd`形式、タイムゾーン: `Asia/Tokyo`）
2. 予約受付リストから、`日付 == 今日` かつ `ステータス == "active"` の行を抽出
3. `開始時間` の昇順でソート
4. 件数に応じてテキストを生成

### 3.4 レスポンス（テキスト — ロボットが読み上げ）

**予約なし**:
```
本日の会議室予約は見つかりませんでした。恐れ入りますが、総務にご確認ください。
```

**予約1件**:
```
本日の会議室予約は、{会議室名}、{開始時間(読み上げ形式)}から{終了時間(読み上げ形式)}、{名前}様と{お客様名}様の{打合せ内容}です。{会議室名}は{場所案内}にございます。
```

例:
```
本日の会議室予約は、第一会議室、10時から11時、田中様と○○株式会社様の製品打合せです。第一会議室は2階の突き当たり右側にございます。
```

**予約複数件**:
```
本日の会議室予約は{N}件あります。{開始時間}から{会議室名}で{名前}様、{開始時間}から{会議室名}で{名前}様、...です。
```

例:
```
本日の会議室予約は3件あります。10時から第一会議室で田中様、13時から応接間1で佐藤様、15時から第一会議室で鈴木様です。
```

### 3.5 時間の読み上げ形式変換

GASコード内で以下の変換を行う:

| スプレッドシート値 | 読み上げ形式 |
|---|---|
| `09:00` | `9時` |
| `09:30` | `9時30分` |
| `10:00` | `10時` |
| `13:30` | `13時30分` |

```javascript
function formatTimeForSpeech(timeStr) {
  // "09:00" → "9時", "09:30" → "9時30分"
  var parts = timeStr.split(':');
  var h = parseInt(parts[0], 10);
  var m = parseInt(parts[1], 10);
  if (m === 0) return h + '時';
  return h + '時' + m + '分';
}
```

### 3.6 会議室の場所案内（コード内に固定定義）

```javascript
var ROOM_LOCATION = {
  '第一会議室': '2階の突き当たり右側',
  '応接間1': '1階ロビー奥',
  '応接間2': '1階ロビー奥'
};
```

※ 実際の場所情報は確認の上で修正してください。

## 4. API #2: スタッフ情報取得 — `getStaffInfo`

### 4.1 リクエスト

```
GET /exec?action=getStaffInfo&staffName=山田花子
```

### 4.2 参照先スプレッドシート（スタッフ管理アプリのデータ）

GAS-Bから `SpreadsheetApp.openById(STAFF_SPREADSHEET_ID)` で読み取り。

**シート名**: `外出ホワイトボード`
**ヘッダー行**: 3行目
**データ行**: 4行目〜19行目（設定値 `DATA_RANGE` に依存）

#### 現在のカラム構成（実機確認済み）

| 列 | ヘッダー | GAS-B での用途 |
|---|---|---|
| A | 人員 | スタッフ名（検索キー） |
| B | 所在地 | 所在地判定（本社工場/新工場/外出/休み） |
| C | 内線 | 内線番号の案内 |
| D | 行先 | 外出時の補足情報（読み上げに含めてもよい） |
| E | 出社日 | （ロボット案内では不使用） |
| F | 帰社時刻 | 外出時の帰社予定案内 |
| G | 備考 | 補足情報 |

#### 追加が必要な列: 部署

**スタッフ管理スプレッドシートに「部署」列を追加**する。追加位置は運用に合わせて柔軟に決定可能だが、以下を推奨:

| 列 | ヘッダー（変更後） |
|---|---|
| A | 人員 |
| **B** | **部署**（新規追加） |
| C | 所在地 |
| D | 内線 |
| E | 行先 |
| F | 出社日 |
| G | 帰社時刻 |
| H | 備考 |

**重要**: 列を挿入する場合、GAS-C（スタッフ管理アプリ）の `SheetService.gs` でのカラムインデックスも合わせて更新が必要。GAS-Cに列追加のマイグレーション関数を追加する。

**代替案（列追加が難しい場合）**: 総務担当者をGAS-Bのコード内に固定配列で定義:
```javascript
var SOUMU_STAFF = ['山田花子', '佐藤一郎'];
```

### 4.3 処理ロジック

1. `staffName`パラメータを受け取る
2. スタッフ管理シートの A列（人員）で完全一致検索
3. **`staffName === "総務"` の場合の特殊処理**:
   - 部署 == "総務" のスタッフを抽出
   - その中から 所在地 が "本社工場" または "新工場"（在席者）を優先
   - 在席者が見つかれば1名を返す、いなければ総務代表内線を案内
4. 該当スタッフの所在地に応じてテキストを生成

### 4.4 レスポンス — 所在地別の案内文（確定）

**本社工場の場合**:
```
{スタッフ名}さんは現在、本社工場におります。内線番号は{内線}番です。受付横の内線電話からおかけください。
```

**新工場の場合**:
```
{スタッフ名}さんは現在、新工場におります。内線番号は{内線}番です。受付横の内線電話からおかけください。
```

**外出の場合**:
```
{スタッフ名}さんは現在外出中です。恐れ入りますが、総務、内線{総務内線}番にご連絡ください。
```

※ `行先`や`帰社時刻`がある場合は補足を追加:
```
{スタッフ名}さんは現在外出中です。{帰社時刻}頃にお戻りの予定です。恐れ入りますが、総務、内線{総務内線}番にご連絡ください。
```

**休みの場合**:
```
{スタッフ名}さんは本日お休みです。恐れ入りますが、総務、内線{総務内線}番に代わりの担当者をお尋ねください。
```

**スタッフが見つからない場合**:
```
申し訳ございません。該当するスタッフが見つかりませんでした。総務、内線{総務内線}番にお問い合わせください。
```

**総務問い合わせで在席者がいる場合**:
```
総務の{スタッフ名}さんは現在、{所在地}におります。内線番号は{内線}番です。受付横の内線電話からおかけください。
```

**総務問い合わせで在席者がいない場合**:
```
申し訳ございません。総務担当者は現在不在です。恐れ入りますが、内線{総務内線}番にメッセージを残していただくか、しばらくお待ちください。
```

## 5. API #3: スタッフ一覧取得 — `getStaffList`

### 5.1 リクエスト

```
GET /exec?action=getStaffList
```

### 5.2 処理ロジック

1. スタッフ管理シートのA列（人員）から全スタッフ名を取得
2. 空行を除外
3. カンマ区切りのテキストで返す

### 5.3 レスポンス

```
山田花子,佐藤一郎,鈴木次郎,田中三郎,...
```

**用途**: RoFlowのメニューにスタッフ一覧を表示する際に使用。動的メニュー表示ができない場合は、RoFlowのメニューに主要スタッフ名を静的に設定し、この API は不使用。

## 6. RoFlowフロー変更仕様

### 6.1 変更概要

NUWA連絡帳モジュール（4箇所）を **APIモジュール + ロボット表現モジュール** の組み合わせに置き換える。

### 6.2 変更後のフロー構成

```
【打合せによる訪問ルート】
開始 → 時間(00:00-23:59) → 来客検知 → 受付メニュー
  |
  ├→ 打合せによる訪問 → 訪問案内(ID00002) → 会議室予約取得API(ID00003)
  │     ├→ 成功 → 予約案内(ID00004) → [END]
  │     ├→ 失敗 → 予約確認エラー対応(ID00005)
  │     │     ├→ スタッフに確認 → [スタッフ情報取得API] → 案内読み上げ → [END]
  │     │     └→ 総務に確認 → [総務情報取得API] → 案内読み上げ → [END]
  │     └→ タイムアウト → 予約確認タイムアウト対応(ID00006)
  │           ├→ スタッフに確認 → [スタッフ情報取得API] → 案内読み上げ → [END]
  │           └→ 総務に確認 → [総務情報取得API] → 案内読み上げ → [END]
  │
  ├→ 社内スタッフに用事 → スタッフ選択メニュー → [スタッフ情報取得API] → 案内読み上げ → [END]
  │
  └→ その他のご用件 → [総務情報取得API] → 案内読み上げ → [END]
```

### 6.3 RoFlow APIモジュール設定（GAS-BのデプロイURL使用）

| API名 | メソッド | URL | パラメータ |
|---|---|---|---|
| 会議室予約取得 | GET | `https://script.google.com/macros/s/{GAS-BのデプロイID}/exec` | `action`=`getTodayReservations` |
| スタッフ情報取得 | GET | 同上 | `action`=`getStaffInfo`, `staffName`=`{{選択されたスタッフ名}}` |
| 総務情報取得 | GET | 同上 | `action`=`getStaffInfo`, `staffName`=`総務` |
| スタッフ一覧取得 | GET | 同上 | `action`=`getStaffList` |

**ポイント**: 全てのRoFlow APIリクエストは **GAS-Bの1つのURL** に集約。`action`パラメータで分岐。

### 6.4 RoFlowモジュール変更一覧

| ID | モジュール名 | 変更内容 |
|---|---|---|
| ID00001 | 受付メニュー | テキスト入力（「ご用件を選択してください」等） |
| ID00003 | 会議室予約取得 | URLをGAS-BのデプロイURLに変更、レスポンス変数名 `reservationInfo` を設定 |
| ID00004 | 予約案内 | 読み上げテキストに `{{reservationInfo}}` を使用 |
| ID00008 | 内線接続 | **削除** → APIモジュール（スタッフ情報取得）+ ロボット表現に置換 |
| ID00011 | 総務呼出 | **削除** → APIモジュール（総務情報取得）+ ロボット表現に置換 |
| ID00012 | エラー時内線接続 | **削除** → APIモジュール（スタッフ情報取得）+ ロボット表現に置換 |
| ID00014 | タイムアウト時内線接続 | **削除** → APIモジュール（総務情報取得）+ ロボット表現に置換 |

## 7. GAS-C（スタッフ管理アプリ）への変更

### 7.1 スプレッドシートに「部署」列を追加

既存の外出ホワイトボードに「部署」列を追加する。

**現在の構成**:
```
A:人員 | B:所在地 | C:内線 | D:行先 | E:出社日 | F:帰社時刻 | G:備考
```

**変更後の構成**:
```
A:人員 | B:部署 | C:所在地 | D:内線 | E:行先 | F:出社日 | G:帰社時刻 | H:備考
```

### 7.2 GAS-Cのマイグレーション

`Setup.gs` に列追加マイグレーション関数を追加し、以下を実行:

1. B列の前に新しい列を挿入（既存B〜G列がC〜H列にシフト）
2. ヘッダー行（3行目）のB列に「部署」を設定
3. ScriptProperties の `HEADER_RANGE` を `A3:H3` に更新
4. ScriptProperties の `DATA_RANGE` を `A4:H19` に更新

### 7.3 GAS-C の SheetService.gs カラムインデックス更新

`getDayData` と `updateRow` / `applyPatch` で参照するカラムインデックスを更新:

| 変更前 | 変更後 |
|---|---|
| `row[0]` = name | `row[0]` = name（変更なし） |
| `row[1]` = location | `row[1]` = department（新規） |
| `row[2]` = extension | `row[2]` = location |
| `row[3]` = destination | `row[3]` = extension |
| `row[4]` = startTime | `row[4]` = destination |
| `row[5]` = endTime | `row[5]` = startTime |
| `row[6]` = note | `row[6]` = endTime |
| — | `row[7]` = note |

## 10. 実装手順（作業順序）

### Phase 1: GAS-C（スタッフ管理アプリ）の修正
