# RoFlow 設定変更指示書（詳細版）

**対象**: NUWA RoFlow 受付プロジェクト（株式会社ラインワークス 新工場事務所）
**作成日**: 2026年3月18日
**目的**: 内線接続（NUWA連絡帳）をAPI経由のスタッフ情報案内に置き換える

---

## 概要

現在のフローでは「社内スタッフに用事」「その他のご用件」選択時にNUWA連絡帳で内線接続を試みるが、内線が使えないため、代わりにGAS APIでスタッフの所在地・内線番号を取得し、ロボットが音声で案内する方式に変更する。

---

## 使用するURL

### ロボット専用API（GAS-B）— 全てのAPIリクエストで使用

```
https://script.google.com/a/macros/lineworks-local.info/s/AKfycbwKpzIUqL5kvn1XTy6PMc11dvILMGmNIaeIpadNCLB4fAia-OM8BQCsF5Ge-eFvB-JR/exec
```

> **重要**: 上記URLは全てのAPIリクエスト（会議室予約・スタッフ情報・スタッフ一覧）で共通です。`action`パラメータで機能を切り替えます。

### 参考：会議室予約アプリ（GAS-A）のURL — 今回は使用しない

```
https://script.google.com/a/macros/lineworks-local.info/s/AKfycbyUKphZ-kTss4o0pEz6j2Q49IAGRiEDIah7501yugZh3yDvVvXnEa78vaMgg4oDR5dMxg/exec
```

> ※ これはWeb UIアプリのURLです。RoFlowのAPI呼び出しには使用しません。

## 作業4: ID00008 内線接続 → スタッフ情報取得APIに置換

### 現在のフロー

```
[スタッフ呼出案内 ID00007] → [NUWA連絡帳: 内線接続 ID00008] → [内線無応答案内 ID00009]
```

### 変更後のフロー

```
[スタッフ呼出案内 ID00007] → [API: スタッフ情報取得（新規）] → 成功 → [ロボット表現: スタッフ情報案内（新規）]
                                                              → 失敗 → [ロボット表現: 取得失敗案内（新規）]
```

### 手順

#### 4-1. ID00008（NUWA連絡帳）を削除

フローからID00008モジュールを削除する。

#### 4-2. 新規APIモジュールを追加: スタッフ情報取得

ID00007（スタッフ呼出案内）の次に、新しいAPIモジュールを追加する。

**RoFlowライブラリにAPI定義を登録（まだ登録していない場合）:**

| 設定項目 | 値 |
|---|---|
| API名 | スタッフ情報取得 |
| リクエスト方式 | GET |
| URL | `https://script.google.com/a/macros/lineworks-local.info/s/AKfycbwKpzIUqL5kvn1XTy6PMc11dvILMGmNIaeIpadNCLB4fAia-OM8BQCsF5Ge-eFvB-JR/exec` |
| パラメータ KEY 1 | `action` |
| パラメータ VALUE 1 | `getStaffInfo` |
| パラメータ KEY 2 | `staffName` |
| パラメータ VALUE 2 | `{{staffName}}` |
| レスポンス変数名 | `staffInfo` |

> **`{{staffName}}`について**: これはRoFlowの変数です。「社内スタッフに用事」を選択した後のメニューで来客が選んだスタッフ名が入ります。RoFlowのメニューモジュールで選択肢をスタッフ名にし、選択結果を変数`staffName`に格納する設定が必要です。

**スタッフ選択メニューの設定（ID00007の後、APIの前に挿入が必要な場合）:**

| 設定項目 | 値 |
|---|---|
| モジュール種類 | メニュー |
| テキスト | `ご用のあるスタッフを選択してください。` |
| 選択肢1 | `山中` |
| 選択肢2 | `木下` |
| 選択肢3 | `辻野` |
| 選択肢4 | `松下` |
| 選択肢5 | `田中(雄)` |
| 選択肢6 | `五十嵐` |
| 選択肢7 | `田中(美)` |
| 選択肢8 | `細野` |
| 選択肢9 | `内山` |
| 選択肢10 | `箱崎` |
| 選択肢11 | `今泉` |
| 選択肢12 | `笹生` |
| 選択肢13 | `岩間` |
| 選択肢14 | `蛭田` |
| 選択肢15 | `日暮` |
| 選択肢16 | `奥` |
| 選択結果の格納変数名 | `staffName` |

> **注意**: RoFlowのメニューに表示できる選択肢数に上限がある場合は、主要スタッフに絞るか、ページ分割を検討してください。

## 作業6: ID00012 エラー時内線接続 → スタッフ情報取得APIに置換

### 現在のフロー

```
[予約確認エラー対応 ID00005] → 「社内スタッフに内線接続する」選択 → [NUWA連絡帳: エラー時内線接続 ID00012]
                             → 「総務に確認してもらう」選択 → [エラー時総務案内 ID00013]
```

### 変更後のフロー

```
[予約確認エラー対応 ID00005] → 「社内スタッフに確認する」選択 → [API: スタッフ情報取得] → 成功 → [ロボット表現: {{staffInfo}}]
                                                                                    → 失敗 → [ロボット表現: 取得失敗案内]
                             → 「総務に確認してもらう」選択 → [API: 総務情報取得] → 成功 → [ロボット表現: {{soumuInfo}}]
                                                                                → 失敗 → [ロボット表現: 総務取得失敗案内]
```

### 手順

#### 6-1. ID00012（NUWA連絡帳）を削除

#### 6-2. 「社内スタッフに内線接続する」の分岐先を変更

作業4で作成した **「スタッフ情報取得」APIモジュール** と同じ構成のモジュールを接続する。

| 接続先 | モジュール構成 |
|---|---|
| スタッフに確認 | [スタッフ選択メニュー] → [API: スタッフ情報取得] → [ロボット表現: `{{staffInfo}}`] |
| 総務に確認 | [API: 総務情報取得] → [ロボット表現: `{{soumuInfo}}`] |

> **注意**: 作業4で作成したスタッフ選択メニュー＋APIモジュールを再利用（同じモジュールに接続）できる場合はそちらを使用してください。RoFlowで同一モジュールの再利用ができない場合は、同じ設定で新規モジュールを作成してください。

#### 6-3. 「総務に確認してもらう」の分岐先を変更

ID00013（エラー時総務案内）を **「総務情報取得」APIモジュール** に置き換えるか、ID00013の前に総務情報取得APIを挿入する。

構成は作業5と同じ:
[API: 総務情報取得] → 成功 → [ロボット表現: `{{soumuInfo}}`]
                   → 失敗 → [ロボット表現: 取得失敗案内]

## 注意事項

1. **URLは全て同じ**: 3つのAPI定義で使用するURLは全て同一です。`action`パラメータで機能が切り替わります。

2. **変数名の大文字小文字**: `reservationInfo`、`staffInfo`、`soumuInfo` の変数名は大文字小文字を正確に入力してください。

3. **総務のパラメータは固定値**: 総務情報取得のstaffNameは`総務`という固定の文字列です。`{{}}` で囲まないでください。

4. **スタッフ選択メニューの名前**: スプレッドシートのA列（人員）に登録されている名前と完全一致させてください。現在のスタッフ一覧: `山中,木下,辻野,松下,田中(雄),五十嵐,田中(美),細野,内山,箱崎,今泉,笹生,岩間,蛭田,日暮,奥`

5. **デプロイURLの更新**: GAS-Bのコードを修正して再デプロイした場合、URLが変わることがあります。その場合はRoFlowのAPI定義も更新が必要です。
