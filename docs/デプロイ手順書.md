## Step 1: GAS-C（スタッフ管理アプリ）のマイグレーション

### 1-1. GASエディタを開く

ブラウザで以下のURLを開きます：
```
https://script.google.com/u/0/home/projects/12VI1i4TK3DmPMkd7ctJ9gV-2bsMbeP7p9mVws4LPscU7fZOC17F-io67/edit
```

### 1-2. 変更が必要なファイル一覧

GASエディタの左側にファイル一覧が表示されます。以下の5ファイルを更新します：

| GASエディタ上のファイル名 | リポジトリのファイル | 操作 |
|---|---|---|
| `Setup.gs` (Setup) | `-inout-board/Setup.gs` | 内容を上書き |
| `SheetService.gs` (SheetService) | `-inout-board/SheetService.gs` | 内容を上書き |
| `index.html` (index) | `-inout-board/index.html` | 内容を上書き |
| `style.html` (style) | `-inout-board/style.html` | 内容を上書き |
| `ui.js.html` (ui.js) | `-inout-board/ui.js.html` | 内容を上書き |

**更新方法（各ファイルで繰り返し）：**
1. GASエディタ左側のファイル名をクリック
2. エディタ内の内容を `Ctrl+A` で全選択
3. リポジトリのファイル内容を `Ctrl+C` でコピー
4. GASエディタに `Ctrl+V` で貼り付け

> **注意**: `Code.gs` と `Validation.gs` は変更不要です。そのままにしてください。

### 1-3. マイグレーション関数を実行

コードを貼り付けた後、**部署列をスプレッドシートに追加**します。

1. GASエディタ上部のドロップダウンで `migrateAddDepartmentColumn` を選択
   ```
   [▶ 実行] のすぐ左にあるドロップダウン → migrateAddDepartmentColumn を選ぶ
   ```
2. `▶ 実行` ボタンをクリック
3. 初回は「承認が必要です」と表示されるので、許可してください
4. 実行ログに以下が表示されれば成功：
   ```
   B列を挿入しました（既存B〜G列がC〜H列にシフト）。
   B3に「部署」ヘッダーを設定しました。
   ScriptProperties を更新しました:
     HEADER_RANGE: A3:H3
     DATA_RANGE: A4:H19
   マイグレーション完了。新しいバージョンをデプロイしてください。
   ```

### 1-4. スプレッドシートで部署データを入力

1. スタッフ管理のスプレッドシートを開く
2. B列（部署）に各スタッフの部署を入力

   例：
   ```
   A列(人員)    B列(部署)    C列(所在地)  D列(内線)  ...
   ─────────────────────────────────────────────────
   山田花子      総務         本社工場      1234      ...
   佐藤一郎      営業         新工場        5678      ...
   鈴木次郎      技術         外出          9012      ...
   ```

   > **「総務」がロボットAPIで使うキーワードです。**
   > 総務担当者には必ず「総務」または「総務部」と入力してください。

### 1-5. GAS-Cを再デプロイ

1. GASエディタ右上の `デプロイ` → `デプロイを管理` をクリック
2. 右上の鉛筆アイコン（編集）をクリック
3. 「バージョン」を `新しいバージョン` に変更
4. 説明に `部署列追加` と入力
5. `デプロイ` ボタンをクリック

### 1-6. 動作確認

WebアプリのURLにアクセスして以下を確認：
- テーブルに「部署」列が表示されている
- 部署列は編集できない（読み取り専用）
- 他の列（所在地・内線・行先 等）は従来通り編集・保存できる

## Step 2: GAS-B（ロボット専用API）の新規作成

### 2-1. 新しいGASプロジェクトを作成

1. ブラウザで https://script.google.com を開く
2. 左上の `＋ 新しいプロジェクト` をクリック
3. プロジェクト名を `ロボット受付API` に変更（左上の「無題のプロジェクト」をクリック）

### 2-2. ファイルを作成してコードを貼り付け

最初から `Code.gs` が1つあります。まず追加のファイルを作成します。

**ファイル追加の操作（4回繰り返し）：**
1. 左側のファイル一覧の `＋` ボタン → `スクリプト` を選択
2. ファイル名を入力（`.gs`は自動で付くので不要）

| 作成順 | 入力するファイル名 | リポジトリのファイル |
|---|---|---|
| (既存) | Code | `robot-api-gas/Code.gs` の内容を貼り付け |
| 1 | Config | `robot-api-gas/Config.gs` の内容を貼り付け |
| 2 | Reservation | `robot-api-gas/Reservation.gs` の内容を貼り付け |
| 3 | Staff | `robot-api-gas/Staff.gs` の内容を貼り付け |
| 4 | TextBuilder | `robot-api-gas/TextBuilder.gs` の内容を貼り付け |

**各ファイルへのコード貼り付け：**
1. GASエディタ左側のファイル名をクリック
2. エディタ内の内容を `Ctrl+A` で全選択
3. リポジトリの対応するファイル内容をコピー
4. GASエディタに貼り付け

> **注意**: `appsscript.json` は手動で編集します（次のステップ）。

### 2-3. appsscript.json を編集

1. GASエディタ左側の歯車アイコン（プロジェクトの設定）をクリック
2. `「appsscript.json」マニフェスト ファイルをエディタで表示する` にチェックを入れる
3. 左側ファイル一覧に `appsscript.json` が表示されるのでクリック
4. 内容を以下に置き換え：

```json
{
  "timeZone": "Asia/Tokyo",
  "dependencies": {},
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8",
  "webapp": {
    "executeAs": "USER_DEPLOYING",
    "access": "ANYONE_ANONYMOUS"
  }
}
```

### 2-4. スクリプトプロパティを設定

1. GASエディタ左側の歯車アイコン（プロジェクトの設定）をクリック
2. 「スクリプト プロパティ」セクションまでスクロール
3. `スクリプト プロパティを追加` を3回クリックして以下を設定：

| プロパティ | 値 | 確認方法 |
|---|---|---|
| `RESERVATION_SPREADSHEET_ID` | 会議室予約スプレッドシートのID | 会議室予約のスプレッドシートURLの `/d/` と `/edit` の間の文字列 |
| `STAFF_SPREADSHEET_ID` | スタッフ管理スプレッドシートのID | 外出記入板のスプレッドシートURLの `/d/` と `/edit` の間の文字列 |
| `SOUMU_EXTENSION` | 総務の代表内線番号 | 例: `1000`（実際の番号に置き換え） |

> **スプレッドシートIDの確認方法：**
> スプレッドシートをブラウザで開き、URLからIDを取得します。
> ```
> https://docs.google.com/spreadsheets/d/【★ここがID★】/edit#gid=0
> ```

4. `スクリプト プロパティを保存` をクリック

### 2-5. テスト実行

デプロイ前にコードが正しく動くかテストします。

1. GASエディタで `Code.gs` を開く
2. 以下のテスト関数をファイル末尾に一時的に追加：

```javascript
/** テスト用（デプロイ前に実行して確認、確認後は削除してOK） */
function testAPIs() {
  // 会議室予約テスト
  Logger.log('=== 会議室予約テスト ===');
  Logger.log(getTodayReservations());

  // スタッフ一覧テスト
  Logger.log('=== スタッフ一覧テスト ===');
  Logger.log(getStaffList());

  // スタッフ情報テスト（実在するスタッフ名に変更してください）
  Logger.log('=== スタッフ情報テスト ===');
  Logger.log(getStaffInfo('山田花子'));

  // 総務テスト
  Logger.log('=== 総務テスト ===');
  Logger.log(getStaffInfo('総務'));
}
```

3. ドロップダウンで `testAPIs` を選択して `▶ 実行`
4. 初回は「承認が必要です」と表示 → 許可
5. 実行ログで自然な日本語テキストが返っていることを確認

   期待される出力例：
   ```
   === 会議室予約テスト ===
   本日の会議室予約は、第一会議室、10時から11時、田中様と○○株式会社様の製品打合せです。第一会議室は2階の突き当たり右側にございます。

   === スタッフ一覧テスト ===
   山田花子,佐藤一郎,鈴木次郎,...

   === スタッフ情報テスト ===
   山田花子さんは現在、本社工場におります。内線番号は1234番です。受付横の内線電話からおかけください。

   === 総務テスト ===
   総務の山田花子さんは現在、本社工場におります。内線番号は1234番です。受付横の内線電話からおかけください。
   ```

6. テスト確認後、`testAPIs` 関数は削除しても残してもOKです

### 2-6. Web Appとしてデプロイ

1. GASエディタ右上の `デプロイ` → `新しいデプロイ` をクリック
2. 左上の歯車アイコン → `ウェブアプリ` を選択
3. 以下を設定：

| 設定項目 | 値 |
|---|---|
| 説明 | `v1.0 ロボット受付API` |
| 次のユーザーとして実行 | `自分` |
| アクセスできるユーザー | `全員` |

4. `デプロイ` をクリック
5. **表示されるURLをコピーして保存してください**

   ```
   https://script.google.com/macros/s/XXXXXXXXXXXXX/exec
   ```

   > **このURLがRoFlowに設定するAPIのURLです。**

### 2-7. ブラウザでAPIテスト

デプロイ後、以下のURLをブラウザに直接入力して動作確認できます：

**会議室予約テスト：**
```
https://script.google.com/macros/s/【デプロイID】/exec?action=getTodayReservations
```
→ 本日の予約情報がテキストで表示される

**スタッフ一覧テスト：**
```
https://script.google.com/macros/s/【デプロイID】/exec?action=getStaffList
```
→ カンマ区切りのスタッフ名が表示される

**スタッフ情報テスト：**
```
https://script.google.com/macros/s/【デプロイID】/exec?action=getStaffInfo&staffName=山田花子
```
→ スタッフの所在地・内線情報がテキストで表示される（「山田花子」は実在するスタッフ名に変更）

**総務テスト：**
```
https://script.google.com/macros/s/【デプロイID】/exec?action=getStaffInfo&staffName=総務
```
→ 総務担当者の情報がテキストで表示される

## Step 3: RoFlowの設定変更

### 3-1. 前提

Step 2-6 で取得したGAS-BのデプロイURLが必要です：
```
https://script.google.com/macros/s/【デプロイID】/exec
```

### 3-2. RoFlowライブラリにAPIを登録

NUWAのRoFlowエディタを開き、**ライブラリのAPIリスト**に以下の定義を追加します。

#### API定義 1: 会議室予約取得（既存ID00003を更新）

| 設定項目 | 値 |
|---|---|
| API名 | 会議室予約取得 |
| メソッド | GET |
| URL | `https://script.google.com/macros/s/【デプロイID】/exec` |
| パラメータ KEY 1 | `action` |
| パラメータ VALUE 1 | `getTodayReservations` |
| レスポンス変数名 | `reservationInfo` |

#### API定義 2: スタッフ情報取得（新規追加）

| 設定項目 | 値 |
|---|---|
| API名 | スタッフ情報取得 |
| メソッド | GET |
| URL | `https://script.google.com/macros/s/【デプロイID】/exec` |
| パラメータ KEY 1 | `action` |
| パラメータ VALUE 1 | `getStaffInfo` |
| パラメータ KEY 2 | `staffName` |
| パラメータ VALUE 2 | `{{staffName}}` ← RoFlowの変数（メニューで選択されたスタッフ名） |
| レスポンス変数名 | `staffInfo` |

#### API定義 3: 総務情報取得（新規追加）

| 設定項目 | 値 |
|---|---|
| API名 | 総務情報取得 |
| メソッド | GET |
| URL | `https://script.google.com/macros/s/【デプロイID】/exec` |
| パラメータ KEY 1 | `action` |
| パラメータ VALUE 1 | `getStaffInfo` |
| パラメータ KEY 2 | `staffName` |
| パラメータ VALUE 2 | `総務` ← 固定値 |
| レスポンス変数名 | `soumuInfo` |

### 3-3. フロー内のモジュールを置き換え

以下の4つの **NUWA連絡帳** モジュールを **APIモジュール + ロボット表現モジュール** に置き換えます。

#### ID00008: 内線接続 → スタッフ情報取得API

```
変更前: [スタッフ呼出案内] → [NUWA連絡帳: 内線接続(ID00008)]
変更後: [スタッフ呼出案内] → [API: スタッフ情報取得] → [ロボット表現: {{staffInfo}} を読み上げ]
```

1. ID00008（NUWA連絡帳）を削除
2. 代わりに「API」モジュールを追加 → 上記「スタッフ情報取得」APIを選択
3. 成功分岐に「ロボット表現」モジュールを追加
   - 読み上げテキスト: `{{staffInfo}}`
4. 失敗分岐に「ロボット表現」モジュールを追加
   - 読み上げテキスト: `申し訳ございません。情報の取得に失敗しました。受付横の内線電話から総務にご連絡ください。`

#### ID00011: 総務呼出 → 総務情報取得API

```
変更前: [その他ご用件案内] → [NUWA連絡帳: 総務呼出(ID00011)]
変更後: [その他ご用件案内] → [API: 総務情報取得] → [ロボット表現: {{soumuInfo}} を読み上げ]
```

1. ID00011を削除
2. 「API」モジュール → 「総務情報取得」APIを選択
3. 成功分岐: ロボット表現 `{{soumuInfo}}`
4. 失敗分岐: ロボット表現 `申し訳ございません。情報の取得に失敗しました。受付横の内線電話から総務にご連絡ください。`

#### ID00012: エラー時内線接続 → スタッフ情報取得API

```
変更前: [予約確認エラー対応] → [スタッフに確認] → [NUWA連絡帳: エラー時内線接続(ID00012)]
変更後: [予約確認エラー対応] → [スタッフに確認] → [API: スタッフ情報取得] → [ロボット表現]
```

ID00008と同じ構成で置き換え。

#### ID00014: タイムアウト時内線接続 → 総務情報取得API

```
変更前: [予約確認タイムアウト対応] → [総務に確認] → [NUWA連絡帳: タイムアウト時内線接続(ID00014)]
変更後: [予約確認タイムアウト対応] → [総務に確認] → [API: 総務情報取得] → [ロボット表現]
```

ID00011と同じ構成で置き換え。

### 3-4. 会議室予約取得モジュール（ID00003）の更新

1. RoFlowフローエディタでID00003（会議室予約取得）を選択
2. URLを GAS-BのデプロイURL に変更
3. レスポンス変数名に `reservationInfo` を設定
4. ID00004（予約案内）のロボット表現テキストを `{{reservationInfo}}` に変更

### 3-5. 受付メニュー（ID00001）の設定

ID00001のメニューモジュールにテキストが未設定の場合、以下を設定：
```
ご用件をお選びください。
```

### 3-6. テスト

全ルートで動作テスト：

| テストケース | 操作 | 期待結果 |
|---|---|---|
| 打合せ訪問（予約あり） | 「打合せによる訪問」選択 | 会議室・時間・担当者名を読み上げ |
| 打合せ訪問（予約なし） | 同上 | 「予約が見つかりません...」を読み上げ |
| スタッフに用事（在席） | 「社内スタッフに用事」→ スタッフ選択 | 所在地・内線番号を読み上げ |
| スタッフに用事（外出） | 同上（外出中のスタッフ） | 外出メッセージ + 総務内線を読み上げ |
| その他ご用件 | 「その他のご用件」選択 | 総務担当者の情報を読み上げ |
| API失敗 | （ネットワーク切断等） | エラー分岐の案内を読み上げ |

## トラブルシューティング

### 「承認が必要です」と表示される
→ 正常です。初回実行時にGoogleアカウントの権限を許可してください。

### ブラウザでAPIにアクセスすると「不明なアクションです」と表示される
→ URLに `?action=getTodayReservations` などのパラメータが付いているか確認。

### 「システム設定エラー：予約スプレッドシートが設定されていません」
→ スクリプトプロパティ `RESERVATION_SPREADSHEET_ID` が未設定。Step 2-4を確認。

### 「該当するスタッフが見つかりませんでした」
→ staffNameの値がスプレッドシートのA列（人員）と完全一致しているか確認（スペースや全角半角に注意）。

### 「総務担当者は現在不在です」
→ スプレッドシートの部署列に「総務」と入力されたスタッフがいないか、全員の所在地が「外出」または「休み」。

### GAS-Cの部署列が表示されない
→ `migrateAddDepartmentColumn()` を実行したか確認。再デプロイが必要。

### デプロイ後にコードを修正した場合
→ コードを修正しただけではデプロイ済みのバージョンには反映されません。
→ `デプロイ` → `デプロイを管理` → 鉛筆アイコン → `新しいバージョン` → `デプロイ` で更新してください。
